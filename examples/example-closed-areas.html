Generation of vector maps from OpenStreetMap data.

Press rectangle button, select rectangle on a map and wait. Note that this tool will work well with small areas. This map style is supposed to work well with tiny areas, such as crossing. area:highway tagging is supposed to be used, otherwise it will not work.

(there is more text below map, you can also see an animation showing how this tool is supposed to work)

Map generator marking private areas without a public access. You can move to any place in the world and generate map from OSM data.

Detects areas blocked by fences, walls, buildings - without public road/footway leading there.

Areas inaccessible to pedestrians are marked as a strong yellow.

Requires mapped barriers, roads/footways to be taken into account must be within range of the rectangle.

If you liked this tool, something was confusing or you are irritated by terrible performance - you are welcomed to comment about it by creating a public issue or by sending me an email. You can also post comment in OSM diary entry.

If you want to help mapping and you are unsure how to start - you can try StreetComplete (an Android app) allowing contributions, with sole requirements being (1) ability to read (2) ability to see things (3) ability to create an OpenStreetMap acount.

If you want to map something (map missing objects, fix mistake) and you are unsure how to start - visit openstreetmap.org, zoom in to interesting area and press "edit" button and follow instructions. You can contact OpenStreetMap community using one of global or local channels listed at community.osm.be.
